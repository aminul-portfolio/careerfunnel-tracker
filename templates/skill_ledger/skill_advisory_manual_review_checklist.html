AI advisory manual review checklist

This checklist is private and advisory only.

This checklist does not save, update, publish, submit, or mutate records.

This checklist does not verify proficiency, certify skills, or predict employer outcomes.

Review evidence manually before using any skill in your CV, LinkedIn, portfolio, or public profile.

A JD signal does not prove proficiency.

JD signal context is advisory only.

A JD signal does not make a skill claim-ready.

Skill gap signals are advisory only.

Learning recommendations are planning aids.

Use VERIFIED skills in public claims only when the linked evidence, sprint reference, and project context have been manually reviewed.

Do not present LEARNING_TARGET, STUDYING, NO_EVIDENCE, or JD-only signals as verified proficiency.

No live AI provider is called from this checklist.

No raw JD text, prompt text, provider response, or AI output is stored or displayed here.

Evidence source review

Confirm the source of the evidence before using a skill claim. Prefer project work, sprint evidence, tested deliverables, screenshots, or prior work experience that you can explain manually.

Sprint reference review

Check that any sprint reference points to completed work and that you understand what was built, tested, reviewed, and closed in that sprint.

Project link review

Review the project context before using the skill publicly. The link should support the claim and should not expose private application data, raw JD text, or unsupported claims.

Evidence level decision

VERIFIED skills may be considered for public claims only after linked evidence, sprint reference, and project context have been manually reviewed.

LEARNING_TARGET, STUDYING, NO_EVIDENCE, and JD-only signals must not be presented as verified proficiency.

JD signal caution

Treat saved JD signal context as planning information only. A JD signal can help identify what to review, but it cannot verify that you have evidence for a public claim.

Public claim decision boundary

Make a public claim only when you can explain the evidence, describe the project context, and answer follow-up questions without relying on generated text or JD signal context.

CV, LinkedIn, portfolio, and public profile caution

Before using a skill in public material, confirm that the wording is accurate, evidence backed, and does not imply external assessment or guaranteed outcomes.

Manual-only review workflow

  1. Open the private advisory page that raised the skill or JD signal.
  2. Review the evidence source, sprint reference, and project link manually.
  3. Decide whether the evidence supports a limited public claim.
  4. Keep learning items and JD-only signals as planning items until evidence exists.
  5. Use conservative wording when adding skills to CV, LinkedIn, portfolio, or public profile material.
